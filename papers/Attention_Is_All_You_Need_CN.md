## 摘要（Abstract）

目前主流的序列转导（sequence transduction）模型都基于复杂的循环神经网络或卷积神经网络，并采用编码器–解码器（encoder–decoder）结构。表现最好的模型还会通过注意力机制把编码器和解码器连接起来。

我们提出一种全新的、简洁的网络架构——**Transformer**，它**完全基于注意力机制**，彻底抛弃了循环结构和卷积结构。

在两个机器翻译任务上的实验表明，这类模型质量更高，同时具备更强的并行能力、训练时间也显著更短。

- 在 **WMT 2014 英→德**翻译任务上，我们的模型取得 **28.4 BLEU**，比当时所有已有结果（包括模型集成）高出超过 2 个 BLEU。
- 在 **WMT 2014 英→法**翻译任务上，我们的模型在 **8 张 GPU 上训练 3.5 天**就达到 **41.8 BLEU** 的单模型最佳成绩，训练成本只是文献中最佳模型的一小部分。

我们还通过将 Transformer 成功应用于英语成分句法分析（不论数据量大小），证明它能很好地泛化到其他任务。

## 1 引言（Introduction）

循环神经网络（RNN），尤其是**长短期记忆网络（LSTM）**[13] 和**门控循环单元（GRU）**[7]，已经在序列建模和转导任务（例如语言建模和机器翻译）中被公认为最先进的方法 [35, 2, 5]。此后大量工作继续推动着循环语言模型和编码器–解码器架构的边界 [38, 24, 15]。

循环模型通常沿着输入和输出序列的**符号位置**来分解计算。将计算步骤与位置对齐后，模型会基于前一个隐藏状态 $h_{t-1}$ 和位置 $t$ 的输入生成当前隐藏状态 $h_t$。这种**天生的顺序性**使得同一训练样本内部无法并行，序列越长问题越严重——因为内存限制让跨样本的批处理也变得困难。近期工作通过**分解技巧**[21] 和**条件计算**[32] 在计算效率上取得显著改进，但**顺序计算的根本约束仍然存在**。

注意力机制已经成为序列建模和转导模型中不可或缺的一部分，允许建模依赖关系时**不考虑它们在输入或输出序列中的距离**[2, 19]。然而，除了少数例外 [27]，这些注意力机制都是**与循环网络配合使用**的。

在本工作中，我们提出 **Transformer**——一种模型架构，它**完全抛弃循环结构**，完全依赖注意力机制在输入和输出之间建立全局依赖关系。Transformer 显著提升了并行能力，仅需在 **8 张 P100 GPU 上训练 12 小时**就能在翻译质量上达到新的最先进水平。

## 2 背景（Background）

减少顺序计算的尝试也体现在 Extended Neural GPU [16]、ByteNet [18] 和 ConvS2S [9] 中，它们都以**卷积神经网络**作为基本构件，对所有输入、输出位置**并行**计算隐藏表示。在这些模型中，把两个任意输入/输出位置的信号关联起来所需的操作数随它们之间的距离增长——ConvS2S 是**线性增长**，ByteNet 是**对数增长**。这让学习远距离依赖变得更困难 [12]。而在 Transformer 中，这个数量被压缩到**常数**，代价是注意力加权位置平均带来的分辨率损失，我们用**多头注意力**（第 3.2 节）来抵消这一点。

**自注意力（self-attention）**，有时也称为**内部注意力（intra-attention）**，是一种把单个序列中不同位置关联起来以计算该序列表示的注意力机制。自注意力已在阅读理解、抽象摘要、文本蕴含、学习任务无关的句子表示等多种任务中成功应用 [4, 27, 28, 22]。

端到端记忆网络基于一种**循环注意力机制**而非与序列对齐的循环结构，在简单语言问答和语言建模任务上表现良好 [34]。

据我们所知，**Transformer 是第一个完全依赖自注意力来计算输入输出表示，而不使用序列对齐的 RNN 或卷积的转导模型**。在接下来的章节中，我们将描述 Transformer，阐述自注意力的动机，并讨论它相对于 [17]、[18]、[9] 等模型的优势。

## 3 模型架构（Model Architecture）

大多数有竞争力的神经序列转导模型都具有**编码器–解码器**结构 [5, 2, 35]。编码器把输入的符号表示序列 $(x_1, \ldots, x_n)$ 映射成连续表示序列 $\mathbf{z} = (z_1, \ldots, z_n)$。给定 $\mathbf{z}$，解码器**逐元素**生成输出符号序列 $(y_1, \ldots, y_m)$。每一步模型都是**自回归**的 [10]——生成下一个符号时会消费之前已生成的符号作为额外输入。

Transformer 沿用这种整体架构，但**编码器和解码器都使用堆叠的自注意力层和逐位置（point-wise）的全连接层**（分别见图 1 的左半和右半）。

### 3.1 编码器和解码器堆叠（Encoder and Decoder Stacks）

**编码器（Encoder）**

编码器由 **N = 6** 层相同的层堆叠而成。每层有两个子层（sub-layer）：

1. **多头自注意力机制**（multi-head self-attention）
2. **简单的逐位置全连接前馈网络**（position-wise fully connected feed-forward network）

我们对两个子层各自使用**残差连接**（residual connection）[11]，再接**层归一化**（layer normalization）[1]。也就是说，每个子层的输出为：

$$\mathrm{LayerNorm}(x + \mathrm{Sublayer}(x))$$

其中 $\mathrm{Sublayer}(x)$ 是子层自身实现的函数。为了让残差连接更方便，**模型中所有子层以及嵌入层都输出维度 $d_{\text{model}} = 512$ 的向量**。

**解码器（Decoder）**

解码器同样由 **N = 6** 层相同的层堆叠而成。除了编码器每层的两个子层外，解码器**插入第三个子层**，对编码器堆叠的输出执行多头注意力。与编码器类似，每个子层都使用残差连接和层归一化。

我们还修改了解码器中的自注意力子层，**防止位置关注到后续位置**。这种掩码（masking）结合**输出嵌入相对位置偏移 1**，确保位置 $i$ 的预测只能依赖于位置小于 $i$ 的已知输出。

### 3.2 注意力（Attention）

注意力函数可以描述为将一个**查询（query）和一组键值对（key-value pairs）映射到一个输出**，其中查询、键、值和输出都是向量。输出是值的加权和，而每个值的权重由查询与对应键的**兼容性函数**计算得出。

#### 3.2.1 缩放点积注意力（Scaled Dot-Product Attention）

我们把具体的注意力机制称为"**缩放点积注意力**"（Scaled Dot-Product Attention，见图 2）。输入由维度 $d_k$ 的查询和键，以及维度 $d_v$ 的值组成。我们计算查询与所有键的点积，除以 $\sqrt{d_k}$，再应用 softmax 得到值的权重。

实际中，我们把一组查询打包成矩阵 $Q$ 同时计算，键和值也分别打包成矩阵 $K$ 和 $V$。输出矩阵按下面公式计算：

$$\mathrm{Attention}(Q, K, V) = \mathrm{softmax}\!\left(\frac{QK^{T}}{\sqrt{d_k}}\right) V \quad (1)$$

两种最常用的注意力是**加性注意力**[2] 和**点积（乘性）注意力**。点积注意力与我们的算法相同，只是少了 $\frac{1}{\sqrt{d_k}}$ 这个缩放因子。加性注意力用一个带单隐层的前馈网络计算兼容性函数。两者理论复杂度相似，但点积注意力**实际更快、更省内存**，因为可以用高度优化的矩阵乘法代码实现。

$d_k$ 较小时两者表现相近；当 $d_k$ 较大时，**不加缩放的点积注意力不如加性注意力** [3]。我们怀疑是 $d_k$ 大时点积的数值量级过大，把 softmax 推到了梯度极小的区域。为抵消这一效应，我们用 $\frac{1}{\sqrt{d_k}}$ 对点积进行缩放。

#### 3.2.2 多头注意力（Multi-Head Attention）

我们发现，与其用 $d_{\text{model}}$ 维的键、值、查询做一次注意力，不如把查询、键、值分别用 $h$ 个不同的、可学习的线性投影映射到 $d_k$、$d_k$、$d_v$ 维，然后**并行**执行 $h$ 次注意力，得到 $d_v$ 维输出，拼接起来再投影一次得到最终值（图 2 右）。

多头注意力让模型可以**同时关注不同表示子空间、不同位置**的信息。单头注意力受平均化限制，无法做到这一点。

$$\mathrm{MultiHead}(Q, K, V) = \mathrm{Concat}(\mathrm{head}_1, \ldots, \mathrm{head}_h)\, W^{O}$$

其中

$$\mathrm{head}_i = \mathrm{Attention}(Q W^{Q}_i,\, K W^{K}_i,\, V W^{V}_i)$$

投影矩阵为参数矩阵：

$$W^{Q}_i \in \mathbb{R}^{d_{\text{model}} \times d_k},\quad W^{K}_i \in \mathbb{R}^{d_{\text{model}} \times d_k},\quad W^{V}_i \in \mathbb{R}^{d_{\text{model}} \times d_v},\quad W^{O} \in \mathbb{R}^{h d_v \times d_{\text{model}}}$$

本文中我们使用 **h = 8 个并行注意力头**。每个头取 $d_k = d_v = d_{\text{model}} / h = 64$。由于每个头维度降低，总计算开销与单头全维度注意力相近。

#### 3.2.3 模型中注意力的应用

Transformer 在三种不同场景使用多头注意力：

- **编码器–解码器注意力层**：查询来自上一层解码器，键和值来自编码器的输出。这让解码器的每个位置都能关注输入序列的所有位置，模拟了典型 seq2seq 模型中的编–解注意力 [38, 2, 9]。
- **编码器自注意力层**：所有键、值、查询都来自同一处——编码器上一层的输出。编码器的每个位置可以关注编码器上一层所有位置。
- **解码器自注意力层**：让解码器的每个位置关注**截至当前为止**的所有位置（包括自身）。需要阻止向左的信息流以保持自回归性质。我们通过在缩放点积注意力的 softmax 输入中**把非法连接对应的值置为 $-\infty$** 来实现掩码（图 2）。

### 3.3 逐位置前馈网络（Position-wise Feed-Forward Networks）

除了注意力子层，编码器和解码器的每一层还包含一个**全连接前馈网络**，对每个位置**独立且相同**地应用。它由两个线性变换和中间一个 ReLU 激活组成：

$$\mathrm{FFN}(x) = \max(0, x W_1 + b_1) W_2 + b_2 \quad (2)$$

虽然不同位置的线性变换相同，但**层与层之间使用不同的参数**。另一种描述方式是把它看作**两个核大小为 1 的卷积**。输入输出维度为 $d_{\text{model}} = 512$，内层维度为 $d_{ff} = 2048$。

### 3.4 嵌入和 Softmax（Embeddings and Softmax）

与其他序列转导模型类似，我们使用**可学习的嵌入**把输入和输出 token 转换为 $d_{\text{model}}$ 维向量。也使用通常的可学习线性变换和 softmax 函数把解码器输出转换为下一 token 的预测概率。在我们的模型中，**两个嵌入层和 pre-softmax 线性变换共享同一份权重矩阵** [30]。在嵌入层中，我们把这些权重乘以 $\sqrt{d_{\text{model}}}$。

### 3.5 位置编码（Positional Encoding）

由于我们的模型既不包含循环也不包含卷积，为了让模型利用序列的顺序信息，我们必须**注入关于 token 在序列中相对或绝对位置的信息**。为此，我们在编码器和解码器堆叠底部的输入嵌入上**加上"位置编码"**。位置编码与嵌入维度相同（$d_{\text{model}}$），因此可以直接相加。位置编码有多种选择——可学习的或固定的 [9]。

本文中我们使用**不同频率的正弦和余弦函数**：

$$\mathrm{PE}_{(pos, 2i)} = \sin\!\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)$$

$$\mathrm{PE}_{(pos, 2i+1)} = \cos\!\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)$$

其中 $pos$ 是位置，$i$ 是维度。也就是说，位置编码的每个维度对应一个正弦曲线，波长从 $2\pi$ 到 $10000 \cdot 2\pi$ 形成**几何级数**。我们选择这个函数是因为我们猜测它能让模型**很容易学会按相对位置关注**——对任意固定偏移 $k$，$\mathrm{PE}_{pos+k}$ 可以表示为 $\mathrm{PE}_{pos}$ 的线性函数。

我们也实验了用**可学习的位置嵌入**[9] 替代，发现两种版本结果几乎相同（见表 3 行 (E)）。我们选择正弦版本，因为它**可能允许模型外推到训练时没见过的更长序列**。

## 4 为什么用自注意力（Why Self-Attention）

本节我们将自注意力层各方面与循环层、卷积层做对比——它们都用于把一个变长符号表示序列 $(x_1, \ldots, x_n)$ 映射成等长的另一序列 $(z_1, \ldots, z_n)$，其中 $x_i, z_i \in \mathbb{R}^d$，例如典型序列转导编码器或解码器中的隐层。支撑我们使用自注意力的动机有三条：

1. **每层的总计算复杂度**
2. **可并行计算量**（以所需的最少顺序操作数衡量）
3. **网络中长距离依赖之间的路径长度**

学习长距离依赖是许多序列转导任务的关键挑战。影响学习这种依赖能力的一个关键因素是**前向和反向信号在网络中必须穿过的路径长度**。输入输出序列任意两个位置之间的路径越短，**越容易学习长程依赖** [12]。因此我们也对比不同层类型组成网络中**任意两个输入输出位置之间的最大路径长度**。

| 层类型 | 每层复杂度 | 顺序操作数 | 最大路径长度 |
|---|---|---|---|
| Self-Attention | $O(n^2 \cdot d)$ | $O(1)$ | $O(1)$ |
| Recurrent | $O(n \cdot d^2)$ | $O(n)$ | $O(n)$ |
| Convolutional | $O(k \cdot n \cdot d^2)$ | $O(1)$ | $O(\log_k(n))$ |
| Self-Attention (restricted) | $O(r \cdot n \cdot d)$ | $O(1)$ | $O(n/r)$ |

> 表 1：不同层类型的最大路径长度、每层复杂度、最少顺序操作数。其中 $n$ 是序列长度，$d$ 是表示维度，$k$ 是卷积核大小，$r$ 是受限自注意力的邻域大小。

如表 1 所示，**自注意力层用常数个顺序执行的操作就把所有位置连起来，而循环层需要 $O(n)$ 个顺序操作**。就计算复杂度而言，当序列长度 $n$ 小于表示维度 $d$ 时，自注意力层比循环层更快——这在机器翻译中最先进模型的句子表示中（word-piece [38]、byte-pair [31]）是最常见的情况。

为了提升涉及**很长序列**的任务的计算性能，自注意力可以限制为**只考虑以各自输出位置为中心、大小为 $r$ 的邻域**。这会把最大路径长度增至 $O(n/r)$。我们计划在未来的工作中进一步研究。

核宽 $k < n$ 的单层卷积无法连接所有输入输出位置对。要做到这一点，需要堆叠 $O(n/k)$ 层卷积（连续核）或 $O(\log_k(n))$ 层（空洞卷积 [18]），从而增加网络中最长路径的长度。卷积层通常比循环层贵 $k$ 倍。可分离卷积 [6] 能显著降低复杂度到 $O(k \cdot n \cdot d + n \cdot d^2)$。即使 $k = n$，可分离卷积的复杂度也等于一个自注意力层加一个逐位置前馈层的组合——这正是我们模型采用的做法。

附带的好处是，**自注意力能让模型更具可解释性**。我们检查了模型的注意力分布，并在附录中展示和讨论了一些例子。**单个注意力头明显学会执行不同的任务**，许多头表现出与句子**句法和语义结构相关**的行为。

## 5 训练（Training）

### 5.1 训练数据和批处理

我们在标准的 **WMT 2014 英德数据集**上训练，约 **450 万**句对。句子用 **byte-pair encoding（BPE）**[3] 编码，源–目标共享词表约 **37000** 个 token。

对于 **英法**，我们使用更大的 **WMT 2014 英法数据集**，**3600 万**句，拆分为 **32000** word-piece 词表 [38]。

句对按近似序列长度**分批**，每个训练批次包含约 **25000 个源 token 和 25000 个目标 token**。

### 5.2 硬件与训练时长

我们在**一台配备 8 张 NVIDIA P100 GPU 的机器**上训练。基础模型（使用本文描述的超参数）每步约 **0.4 秒**，总共训练 **10 万步 / 12 小时**。

大模型（见表 3 底部）每步 **1.0 秒**，训练 **30 万步（3.5 天）**。

### 5.3 优化器

我们使用 **Adam** 优化器 [20]，$\beta_1 = 0.9$，$\beta_2 = 0.98$，$\epsilon = 10^{-9}$。训练过程中学习率按公式变化：

$$\mathrm{lrate} = d_{\text{model}}^{-0.5} \cdot \min(\mathrm{step\_num}^{-0.5},\, \mathrm{step\_num} \cdot \mathrm{warmup\_steps}^{-1.5}) \quad (3)$$

这相当于在前 $\mathrm{warmup\_steps}$ 步**线性增加**学习率，之后按步数的**反平方根**成比例减小。我们设 $\mathrm{warmup\_steps} = 4000$。

### 5.4 正则化（Regularization）

训练中我们采用**三种正则化**：

- **残差 Dropout**：对每个子层的输出（在加到子层输入并归一化之前）应用 dropout [33]。另外对编码器和解码器堆叠中**嵌入与位置编码之和**也应用 dropout。基础模型取 $P_{\text{drop}} = 0.1$。
- **标签平滑（Label Smoothing）**：训练中我们使用 $\epsilon_{ls} = 0.1$ 的标签平滑 [36]。这会损害 perplexity（因为模型变得更"不确定"），但**能提升准确率和 BLEU**。

在 **WMT 2014 英德**翻译任务上，**大 Transformer 模型**比之前所有已报告模型（包括集成）高出 **2.0 BLEU** 以上，**创下 28.4 BLEU 的新 SOTA**。该模型配置见表 3 最后一行。训练耗时 **3.5 天，8 张 P100 GPU**。即使我们的**基础模型也超过了所有已发表模型和集成**，而训练成本只是任何竞争模型的一小部分。

在 **WMT 2014 英法**任务上，我们的大模型取得 **41.0 BLEU**，超过所有已发表的单模型，**训练成本不到之前 SOTA 模型的 1/4**。英法大模型使用 $P_{\text{drop}} = 0.1$（而非 0.3）。

基础模型用**最后 5 个 checkpoint 的平均**（每 10 分钟写一次）；大模型用**最后 20 个 checkpoint** 的平均。推理使用 **beam search**，beam 大小 4，长度惩罚 $\alpha = 0.6$ [38]。这些超参是在开发集上实验挑选的。推理时最大输出长度 = 输入长度 + 50，但能提前终止则提前终止 [38]。

### 6.3 英语成分句法分析（English Constituency Parsing）

为了验证 Transformer 能否泛化到其他任务，我们在英语成分句法分析上做了实验。这个任务有特殊挑战：输出受到强结构约束约束，且输出长度**显著长于输入**。此外，RNN seq2seq 模型在小数据规模下无法取得 SOTA 结果 [37]。

我们在 **WSJ（Penn Treebank 的一部分）**[25] 上训练了一个 **4 层 Transformer**，$d_{\text{model}} = 1024$，训练数据约 **4 万**句。我们还在半监督设置下训练，使用更大的高置信度与 BerkleyParser 语料（约 **1700 万**句）[37]。WSJ-only 设置使用 **16K** token 词表，半监督设置使用 **32K**。

我们只做了少量实验来选择 dropout（注意力和残差两部分，见 5.4 节）、学习率和 beam 大小（基于 Section 22 开发集），其他参数保持英德基础翻译模型不变。推理时最大输出长度 = 输入长度 + 300。WSJ-only 和半监督设置都使用 beam 大小 **21**，$\alpha = 0.3$。

我们在表 4 中展示了结果：**尽管缺少任务特定的调优，我们的模型表现得出奇地好**，除了 Recurrent Neural Network Grammar [8] 之外，**优于所有已报告模型**。

与 RNN seq2seq 模型 [37] 不同的是，**即使仅在 4 万句的 WSJ 训练集上训练，Transformer 也超过了 BerkeleyParser** [29]。

## 7 结论（Conclusion）

本文中我们提出了 **Transformer**——**首个完全基于注意力的序列转导模型**，用多头自注意力替代了编码器–解码器架构中最常用的循环层。

对于翻译任务，Transformer **比基于循环或卷积的架构训练速度显著更快**。在 **WMT 2014 英德**和 **WMT 2014 英法**翻译任务上，我们都达到了**新的 SOTA**。在前一项任务上，我们的最佳模型甚至**超过了所有之前报告的集成模型**。

我们对基于注意力的模型的未来感到兴奋，并计划将它们应用到其他任务。我们计划将 Transformer 扩展到文本之外的输入输出模态（如图像、音频、视频），并研究**局部的、受限的注意力机制**以高效处理大输入大输出。**让生成过程不那么顺序化**也是我们的研究方向。

我们用于训练和评估模型的代码已在 https://github.com/tensorflow/tensor2tensor 公开。

|---|---|---|
| (1) | 缩放点积注意力 | $\mathrm{Attention}(Q, K, V) = \mathrm{softmax}\!\left(\frac{QK^{T}}{\sqrt{d_k}}\right) V$ |
| (2) | 逐位置前馈网络 | $\mathrm{FFN}(x) = \max(0, x W_1 + b_1) W_2 + b_2$ |
| (3) | 学习率调度 | $\mathrm{lrate} = d_{\text{model}}^{-0.5} \cdot \min(\mathrm{step\_num}^{-0.5}, \mathrm{step\_num} \cdot \mathrm{warmup\_steps}^{-1.5})$ |

## 核心结论速查（Key Takeaways）

1. **Transformer 完全抛弃 RNN/CNN**，仅靠注意力机制实现序列转导。
2. **自注意力让任意两个位置的路径长度是常数 $O(1)$**，而 RNN 是 $O(n)$，CNN 是 $O(\log_k(n))$。
3. **多头注意力（h=8）** 让模型同时关注不同子空间；缩放因子 $\sqrt{d_k}$ 防止 softmax 梯度消失。
4. **正弦位置编码** 给模型注入顺序信息，允许外推到训练时没见过的长度。
5. **残差 + LayerNorm + 前馈** 是每个子层的标配。
6. **共享 embedding 与 pre-softmax 权重** 并乘以 $\sqrt{d_{\text{model}}}$。
7. **Adam + warmup + 反平方根衰减** 学习率调度是标配。
8. **解码器用掩码自注意力** 保证自回归性质。

## 一个最小 PyTorch 复现（Mini PyTorch Implementation）

下面是一个**可直接运行的简化版**缩放点积注意力 + 多头注意力实现，对应论文公式 (1) 和 §3.2.2：

```python
import math
import torch
import torch.nn as nn
import torch.nn.functional as F


class ScaledDotProductAttention(nn.Module):
    """
    论文公式 (1)：
        Attention(Q, K, V) = softmax(Q K^T / sqrt(d_k)) V
    """
    def forward(self, Q, K, V, mask=None):
        # Q: (B, h, n, d_k), K: (B, h, n, d_k), V: (B, h, n, d_v)
        d_k = Q.size(-1)
        # 1) Q 与 K^T 做点积，并除以 sqrt(d_k)
        scores = torch.matmul(Q, K.transpose(-2, -1)) / math.sqrt(d_k)
        # 2) 可选 mask：把不允许关注的位置置为 -inf
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        # 3) softmax 得到权重
        attn = F.softmax(scores, dim=-1)
        # 4) 加权求和 V
        return torch.matmul(attn, V), attn


class MultiHeadAttention(nn.Module):
    """
    论文 §3.2.2 多头注意力
    """
    def __init__(self, d_model=512, h=8):
        super().__init__()
        assert d_model % h == 0, "d_model 必须能被 h 整除"
        self.d_model = d_model
        self.h = h
        self.d_k = d_model // h  # 论文取 d_k = d_v = d_model / h = 64

        # 4 组可学习线性投影：W^Q, W^K, W^V, W^O
        self.W_q = nn.Linear(d_model, d_model)
        self.W_k = nn.Linear(d_model, d_model)
        self.W_v = nn.Linear(d_model, d_model)
        self.W_o = nn.Linear(d_model, d_model)

        self.attn = ScaledDotProductAttention()

    def forward(self, Q, K, V, mask=None):
        B = Q.size(0)

        # 1) 线性投影；2) 拆成 h 个头
        Q = self.W_q(Q).view(B, -1, self.h, self.d_k).transpose(1, 2)  # (B, h, n, d_k)
        K = self.W_k(K).view(B, -1, self.h, self.d_k).transpose(1, 2)
        V = self.W_v(V).view(B, -1, self.h, self.d_k).transpose(1, 2)

        # 3) 并行做 h 次缩放点积注意力
        x, attn = self.attn(Q, K, V, mask=mask)

        # 4) 拼接所有头；5) 最后线性投影 W^O
        x = x.transpose(1, 2).contiguous().view(B, -1, self.d_model)
        return self.W_o(x)


# ====== 自测 ======
if __name__ == "__main__":
    torch.manual_seed(0)
    B, n, d_model = 2, 5, 512
    x = torch.randn(B, n, d_model)
    mha = MultiHeadAttention(d_model=512, h=8)
    out = mha(x, x, x)  # 自注意力：Q=K=V=x
    print("输入形状 :", x.shape)
    print("输出形状 :", out.shape)
    assert out.shape == x.shape
    print("✓ MultiHeadAttention 自检通过")
```
